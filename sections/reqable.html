Reqable is a powerful cross-platform HTTP debugging proxy that allows you to intercept, inspect, and modify HTTP/HTTPS traffic between clients and servers.

What is Reqable?

Reqable is a modern alternative to tools like Charles Proxy and Fiddler. It provides a clean interface for analyzing web traffic, debugging APIs, and testing web applications. With Reqable, you can:

  • Monitor all HTTP/HTTPS traffic in real-time
  • Intercept and modify requests/responses
  • Debug mobile applications
  • Test API endpoints
  • Analyze website performance
  • Support for HTTP/2 and HTTP/3

Android Certificate Installation

To analyze HTTPS traffic from Android devices, you need to install Reqable's certificate:

1

Access Certificate Management

In the Reqable app, navigate to the Tabs section and enable "Certificate Management" under Remote Devices to access certificate controls.

2

Install Root Certificate

In Certificate Management, check the installation status. If not installed, proceed to install the root certificate to your local machine for traffic interception.

3

Export Certificate File

Export the root certificate in your preferred format (.crt, .pem, or .0) from the Export Root Public Certificate options. The exported file contains only the X509 public certificate without storage password.

4

Install Certificate on Android System

On your Android device, go to Settings > Security > More security settings > Credential storage > Install from phone storage and SD card. Select "CA certificate" and choose the certificate file you exported.

5

Verify Certificate Installation

After installation, verify the certificate appears in Trusted Credentials under Credential Storage settings. You can also check Certificate Management in Reqable to confirm the root certificate is properly installed.

Using Reqable for Web Scraping Analysis

Step 1: Start Capturing Traffic

Click the "Start" button in Reqable to begin capturing HTTP traffic. All requests from your configured devices will now appear in the session list.

Step 2: Analyze Requests

Click on any request in the session list to view its details:

  • Headers: View request and response headers
  • Body: Inspect request and response payloads
  • Timing: Analyze request/response timing
  • Raw: View raw HTTP data
  • Hex: View binary data in hexadecimal format

Step 3: Identify API Endpoints

Look for patterns in the captured traffic to identify:

  • API endpoints and their structure
  • Required headers and authentication
  • Request parameters and payload formats
  • Response data structure
  • Pagination and data loading patterns

Step 4: Extract Data for Scraping

Use the information gathered to build your web scraper:

// Based on Reqable analysis, we found this API endpoint
const apiUrl = 'https://api.example.com/data';
const headers = {
  'Authorization': 'Bearer token_from_analysis',
  'Content-Type': 'application/json',
  'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64)'
};

// Make the API request
async function fetchData() {
  const response = await axios.get(apiUrl, {
    headers,
    params: {
      page: 1,
      limit: 100
    }
  });
  return response.data;
}

Advanced Features

Request/Response Modification

Reqable allows you to intercept and modify requests and responses in real-time:

  • Change request headers before they're sent
  • Modify request body data
  • Alter response data before it reaches the client
  • Simulate different server responses
  • Inject custom JavaScript code

Breakpoints

Set breakpoints to pause request/response processing and manually inspect or modify data:

// In Reqable, you can set breakpoints for specific URLs
// This allows you to inspect and modify data in real-time
// Useful for debugging complex API interactions

Scripting Support

Reqable supports JavaScript scripting for advanced automation:

// Example script to modify requests
function onRequest(context, request) {
  // Add custom header to all requests
  request.header('X-Custom-Header', 'MyValue');
  return request;
}

function onResponse(context, response) {
  // Modify response data
  if (response.header('Content-Type').includes('json')) {
    const data = response.json();
    data.modified = true;
    response.body = JSON.stringify(data);
  }
  return response;
}

Best Practices for Web Scraping

Filtering and Search

Use Reqable's powerful filtering to focus on relevant traffic:

  • Filter by domain or URL pattern
  • Search for specific headers or body content
  • Use regular expressions for complex patterns
  • Save filter presets for different projects

Session Management

Organize your analysis sessions effectively:

  • Save important sessions for future reference
  • Export session data for team collaboration
  • Use workspace to organize multiple projects
  • Clear session data regularly to improve performance

Performance Tips

  • Use filters to reduce noise and focus on relevant traffic
  • Enable compression for large session files
  • Use breakpoints selectively to avoid performance impact
  • Clear session history regularly
  • Export configuration for quick setup on different machines
